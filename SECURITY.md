# Security Policy

This repository is public and must only contain public-safe skill specifications, workflows, examples, and adapter logic.

## Reporting Security Issues

Do not open a public issue for secrets, private data exposure, or sensitive workflow leakage. Report privately to the repository maintainers.

## Public Repository Rules

Do not commit:

- API keys, tokens, secrets, credentials, or connection strings.
- Company-internal workflows, templates, data, or customer examples.
- Paid data source integration logic.
- Finance, stock, or investment strategy implementation details.
- Any `internal-only`, `private`, or `confidential` actual content.

## Skill Safety

Public skills should default to read-only analysis unless their purpose explicitly requires safe edits. Destructive actions are prohibited by default and must not be enabled in public example skills.
