# Report Writer

Read `references/productivity-evidence-and-file-safety.md` before using sources or creating files. Use `assets/report-template.md` when the user has not supplied a structure.

1. Confirm the objective, audience, decision or use case, scope, tone, length, deadline, source set, and requested format. Ask only about choices that materially change the report.
2. Build an evidence ledger. Label facts, quotations, inferences, recommendations, and unknowns; preserve source locations, dates, units, and citation identifiers.
3. Select only useful sections, such as executive summary, background, method, findings, analysis, risks, recommendations, and appendices. Do not force every section into every report.
4. Draft conclusions proportionate to the evidence. Surface limitations and conflicting evidence instead of filling gaps.
5. Check terminology, figures, dates, units, cross-references, citations, and recommendation ownership for internal consistency.
6. Return Markdown by default. Generate DOCX or PDF only when the user requests it and the platform provides creation plus validation tools. Write to a new output path unless overwrite is explicit; report any unrendered or unverified checks.

Do not turn an open-ended evidence-gathering request into a completed research claim; hand that work to `research-brief`.

# Output Contract

1. Report objective, audience, scope, and format
2. Structured report with evidence-aware claims
3. Source and evidence notes
4. Limitations, unknowns, and conflicting material
5. Consistency checks and file-validation status

# 报告撰写

使用来源或创建文件前读取 `references/productivity-evidence-and-file-safety.md`。用户未提供结构时使用 `assets/report-template.md`。

1. 确认目标、读者、决策或用途、范围、语气、篇幅、截止时间、材料和输出格式；只有会实质改变报告的选择才询问。
2. 建立证据台账，区分事实、原文引用、推断、建议和未知项，保留位置、日期、单位和引用标识。
3. 按需要选择摘要、背景、方法、发现、分析、风险、建议和附录，不强制套用所有章节。
4. 让结论与证据强度匹配；材料不足或互相冲突时明确限制，不自行补齐。
5. 检查术语、数字、日期、单位、交叉引用、引文和建议责任的一致性。
6. 默认输出 Markdown。仅在用户要求且平台具备创建和验证能力时生成 DOCX/PDF；除非明确授权覆盖，否则写入新路径，并报告未渲染或未验证项目。

开放式证据搜集应转交 `research-brief`，不得伪装成已经完成的实时研究。

# 输出契约

1. 报告目标、读者、范围和格式
2. 基于证据的结构化报告
3. 来源与证据说明
4. 限制、未知项和冲突材料
5. 一致性检查及文件验证状态
