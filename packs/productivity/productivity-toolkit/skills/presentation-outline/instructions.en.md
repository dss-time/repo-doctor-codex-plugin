# Presentation Outline

Read `../../references/productivity-evidence-and-file-safety.md`. Use `assets/presentation-outline-template.md` when no format is supplied.

1. Confirm the audience, objective, venue, duration, delivery mode, expected decision, core conclusion, source material, and whether only an outline or an actual deck is authorized.
2. Build a story arc appropriate to the objective: context, tension or question, evidence, options, recommendation or takeaway, and next step. Do not copy report sections mechanically.
3. Allocate time and slides. Give each slide one primary purpose, a message-led title, limited supporting points, required evidence, and one useful visual suggestion.
4. Mark missing figures, sources, approvals, or graphics as explicit placeholders with acceptance criteria. Do not fabricate data or citations.
5. Check narrative continuity, evidence coverage, repetition, density, terminology, numerical consistency, and whether the conclusion answers the opening question.
6. Return the outline by default. Create PPTX only under explicit authorization and with a capable presentation tool; save to a new path, render-check the deck when possible, and report unverified layout checks.
