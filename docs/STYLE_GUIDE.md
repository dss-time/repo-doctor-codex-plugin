# Style Guide

## Writing Skills

- Use direct workflow instructions.
- Prefer evidence-based output.
- Keep safety boundaries explicit.
- Avoid vague claims such as "be careful" without a checkable action.
- Keep public examples generic.
- Do not include company-specific or customer-specific details.

## Bilingual Writing

- Chinese should be natural, not mechanical translation.
- English should be concise and operational.
- Technical identifiers should stay unchanged.
- Permissions and risk rules must match across languages.

## Output Formats

Use tables for review findings, risk assessments, and validation plans. Require source references such as file paths, page numbers, paragraph numbers, sheet names, or field names when applicable.
