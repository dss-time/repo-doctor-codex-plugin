# Public / Private Boundary

This repository is public. It must only contain generic, low-risk, public-safe material.

## Allowed in This Public Repository

- Skill standards
- Generic workflows
- Platform adapters
- Repo Doctor base pack
- Basic document, Excel, PDF, and Word review skills
- Examples and testing framework
- Finance skill interface specifications
- Finance safety boundary documents

## Not Allowed in This Public Repository

- Company-internal templates
- Company-internal workflows
- Customer cases
- Private data sources
- API tokens, keys, secrets, or credentials
- Financial analysis strategies
- Stock screening rules
- Buy, sell, or hold recommendation logic
- Valuation model weights
- Technical indicator strategy combinations
- Real stock pools
- Investment portfolio strategy
- Any sensitive business rules
- Any actual `internal-only`, `private`, or `confidential` content

## Recommended Split

Use this public repository for framework, schema, adapters, and public-safe base packs. Put private implementations in separate private repositories or private distribution packages.
